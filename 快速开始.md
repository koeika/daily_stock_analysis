## 🚀 三种使用方式

### 1️⃣ 手动触发（推荐先用这个测试）

```bash
# 使用快速启动脚本（推荐）
./start.sh

# 或直接运行命令
python3 main.py              # 完整分析
python3 main.py --market-review  # 仅大盘复盘
```

**特点**:
- ✅ 立即执行,立即看到结果
- ✅ 适合测试和按需分析
- ✅ 分析完成后自动推送到飞书

---

### 2️⃣ 本地定时任务

**步骤1**: 修改 `.env` 配置

```bash
# 启用定时任务
SCHEDULE_ENABLED=true

# 每天18:00执行
SCHEDULE_TIME=18:00
```

**步骤2**: 启动定时服务

```bash
# 前台运行（测试用）
python3 main.py --schedule

# 后台运行（推荐）
nohup python3 main.py --schedule > schedule.log 2>&1 &

# 查看日志
tail -f schedule.log
```

**特点**:
- ✅ 每天自动执行
- ⚠️ 需要电脑保持运行
- ✅ 本地控制,灵活配置

---

### 3️⃣ GitHub Actions 云端定时（最推荐）

**步骤1**: Fork 本仓库到你的 GitHub

**步骤2**: 配置 GitHub Secrets

```
Settings → Secrets and variables → Actions
```

需要配置（从你的 .env 复制）:
- `OPENAI_API_KEY`
- `OPENAI_BASE_URL`
- `OPENAI_MODEL`
- `FEISHU_WEBHOOK_URL`
- `FEISHU_WEBHOOK_SECRET`
- `STOCK_LIST`

**步骤3**: 启用 Actions

```
Actions → Enable workflows
```

**特点**:
- ✅ 零成本,完全免费
- ✅ 无需本地运行
- ✅ 每周一到五 18:00 自动执行
- ✅ 可随时手动触发

## ❓ 常见问题

**Q: 如何立即测试一次?**
```bash
python3 main.py
```

**Q: 如何修改推送时间?**
- 本地: 修改 `.env` 中的 `SCHEDULE_TIME`
- GitHub Actions: 修改 `.github/workflows/daily_analysis.yml`

**Q: 推送失败怎么办?**
1. 检查 Webhook URL 是否正确
2. 查看日志: `tail -f logs/stock_analysis_*.log`
3. 运行测试: `python3 test_feishu_simple.py`

**Q: 想要每天自动推送,但不想电脑一直开着?**
使用 GitHub Actions (方式3),云端自动执行!
